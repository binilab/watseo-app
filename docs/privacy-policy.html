왔어

개인정보처리방침

시행일: 2026년 6월 21일

“왔어”(이하 “서비스”)는 상세 위치 없이 도착 상태만 함께 확인하는 귀가 확인 앱입니다. 본 방침은 서비스가 어떤 정보를 수집하고, 어떤 목적으로 이용하며, 어떻게 관리하는지를 안내합니다. 서비스는 이동 경로나 실시간 위치를 계속 공유하지 않고, 도착 확인 상태와 필요한 알림만 전달하는 것을 목표로 합니다.

1. 수집하는 정보

서비스는 기능 제공에 필요한 최소한의 정보를 수집합니다.

구분항목
계정 정보이메일 주소, 닉네임, 계정 식별자
연결 정보사용자가 직접 맺은 연결 관계 정보
도착 장소사용자가 등록한 도착 장소명
귀가 활동귀가 상태, 도착 확인 기록, 시간 연장 요청 기록, 도움 요청 기록
알림앱 푸시 알림을 위한 기기 토큰 또는 푸시 토큰

서비스는 상세 주소, 좌표, 이동 경로, 실시간 위치를 수집하거나 저장하지 않습니다.

2. 카메라 권한

  • 카메라 권한은 도착 장소의 QR 코드를 스캔하기 위해서만 사용됩니다.
  • 사진이나 동영상을 저장하거나 별도로 보관하는 용도로 사용하지 않습니다.
  • 카메라 권한을 허용하지 않아도 QR 코드 값을 직접 입력해 도착 확인을 진행할 수 있습니다.

3. 위치 정보

  • 현재 서비스는 실시간 위치 공유 기능을 제공하지 않습니다.
  • 이동 경로를 저장하거나 화면에 표시하지 않습니다.
  • 상세 위치는 계속 공유되지 않으며, QR 도착 확인을 통한 도착 상태와 필요한 알림만 전달됩니다.

4. 이용 목적

  • 계정 식별 및 로그인 상태 관리
  • QR 기반 도착 확인 기능 제공
  • 사용자가 연결한 사람에게 필요한 도착 상태 전달
  • 앱 안정성 및 품질 개선
  • 앱 푸시 알림 제공 또는 향후 제공을 위한 준비

5. 제3자 제공 및 처리 위탁

서비스는 다음의 경우에 한해 정보가 외부로 전달될 수 있습니다.

사용자가 연결한 사람에게 전달

사용자가 직접 연결한 사람에게 도착 상태와 필요한 알림(예: 귀가 시작, 도착 확인, 시간 연장 요청, 도움 요청)이 전달될 수 있습니다. 이때에도 상세 위치나 이동 경로는 전달되지 않습니다.

앱 운영을 위한 외부 서비스

서비스 운영을 위해 다음과 같은 외부 서비스가 사용될 수 있습니다.

  • Supabase — 계정 인증 및 데이터 저장
  • Firebase Cloud Messaging(FCM) — 앱 푸시 알림 전송(준비 중 또는 향후 지원 예정)
  • Expo / EAS — 앱 빌드 및 푸시 알림 전달

각 외부 서비스는 해당 기능 제공에 필요한 범위에서만 정보를 처리합니다.
